# Configuration

## Compiler Options

The URUS compiler (`urusc`) is configured through command-line flags. There is no separate config file.

### CLI Flags

| Flag | Default | Description |
|------|---------|-------------|
| `--emit-c` | off | Output generated C to stdout without compiling |
| `-o <name>` | input file name | Output binary name |
| `--help` | - | Show help |

### Examples

```bash
# Default: compile to binary with the same name
urusc program.urus
# → produces program (Linux) or program.exe (Windows)

# Custom output name
urusc program.urus -o myapp

# Only generate C (for debugging)
urusc --emit-c program.urus
```

## Environment Variables

URUS does not use environment variables directly. However, the compiler depends on:

| Variable | Used By | Notes |
|----------|---------|-------|
| `PATH` | OS | Must contain the location of `gcc` and optionally `urusc` |
| `CC` | Makefile | Override C compiler (default: `gcc`) |
| `CFLAGS` | Makefile | Additional compiler flags |

### Override C Compiler

```bash
# Use clang as backend
CC=clang make

# Or manually
clang -std=c11 -o program _urus_tmp.c -lm
```

## Build Configuration

### Makefile Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `CC` | `gcc` | C compiler |
| `CFLAGS` | `-Wall -Wextra -std=c11 -g` | Compiler flags |
| `TARGET` | `urusc` | Output binary name |

### Debug vs Release

Currently URUS always builds with `-g` (debug symbols). For a release build:

```bash
# Manual release build
gcc -Wall -Wextra -std=c11 -O2 -DNDEBUG -Iinclude -o urusc \
    src/main.c src/lexer.c src/ast.c src/parser.c \
    src/util.c src/sema.c src/codegen.c -lm
```

| Mode | Flags | Notes |
|------|-------|-------|
| Debug | `-g` | Default, includes debug symbols |
| Release | `-O2 -DNDEBUG` | Optimized, no assertions |

## Generated C Configuration

The C files generated by URUS use:
- Standard: C11 (`-std=c11`)
- Math library: `-lm` (for `sqrt`, `abs`, etc.)
- Runtime: `urus_runtime.h` (included automatically)

### Temp File

The compiler writes a temporary C file `_urus_tmp.c` in the working directory, then deletes it after GCC finishes. This file can be inspected if compilation fails.

## Import Path Resolution

Imports are resolved relative to the location of the importing file:

```urus
// If this file is at /project/src/main.urus
import "utils.urus";       // → /project/src/utils.urus
import "lib/math.urus";    // → /project/src/lib/math.urus
```

- No search paths or library paths
- Circular imports are detected and produce an error
- Maximum 64 imports per program
